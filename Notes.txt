Constraints  --> Set of rules 
-------------------------------
>> Rule applied to a column or table to maintain data integrity.

PRIMARY KEY ---> unique and not null. to identify each record uniquely. only one primary key column in a table.

AUTO_INCREMENT ---> It increments numeric value.

NOT NULL ---> It ensures no null values are in a col

UNIQUE ---> It ensures there are no duplicates in a col.

CHECK --> It is used to check a condition

DEFAULT --> Adding default value to a column

FOREIGN KEY---> Creates relationship between 2 tables. referential integrity.
Ex: foreign key (name) references table(key);

Joins 
---------
data is normalized(split into multiple tables).

Inner Join
------------
>> Returns only matching records from both tables.
>> Non-matching rows are ignored.
Ex: select (col details) from tab1 inner join tab2 on relationship;

Left Outer Join 
----------------
>> Returns all records from left table.
>> joins records from right.
Ex: select (col details) from tab1 left join tab2 on relationship;


Right Outer Join 
-------------------
>> Returns all records from right table.
>> joins records from left.
Ex: select (col details) from tab1 right join tab2 on relationship;

Full Outer Join 
----------------
>> Returns all records from both tables.
select (col details) from tab1 right join tab2 on relationship 
union
select (col details) from tab1 right join tab2 on relationship;

Cross Join 
-----------
>> Every records from left table will be matched with every records from right table.

rows at result --> rows from left * rows from right
Ex: select * from tab1 cross join tab2;

 Ex: select aggregate fnc .... group by col;

Subqueries 
-------------
>> It is a query inside another query.
Ex: main query (inner query)

Single row Subqueries
----------------------
>> It will return only one row.
Ex: select * from students where 
mark>(select avg(mark) from students);

Multi rows Subqueries 
---------------------
>> It will return Multiple rows.
Ex: select * from students 
where 
mark > ALL (select mark from students where dept="ECE")

ALL 
---
every value from inner query should meet the condition

ANY 
-----
any one value from inner query should meet the condition

Correlated Subqueries 
--------------------
>> A Correlated Subquery depends on the outer query.
Ex: select * from students s1
 where mark > 
 (select avg(mark) from students s2 where dept=s1.dept);
